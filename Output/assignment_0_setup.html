2. What a conda environment is and why it is useful

A conda environment is a workspace for installing specific packages, libraries, etc., for a project. This environment can be shared and is self-contained, meaning that whatever one does in that environment will not affect the system as a whole or other environments.

This is useful by aiding in organization within said project, assuring that it is reproducible between people and time points! This makes collaboration easier while ensuring that no projects interfere with each other or break code due to updating packages, libraries, etc.

Container types are lists, tuples, sets, and dictionaries. Lists can contain any type of data, such as integers, floats, strings, mix and match, etc. Just like strings, these are iterable, meaning we can access indexes through code like int_list[2]. Lists can be nested x = [[1,2,3],[4,5,6]], or added to with the "append" code. These are mutable, meaning you can assign a new value to a certain index.

Tuples are similar in that they are iterable, but they are not mutable. Sets are also similar to lists, but cannot contain duplicate elements. Dictionaries (discussed in the second notebook) are another type of data structure that maps values to keys. These are mutable, but unlike lists and tuples, are unordered. To access a part of a dictionary, the key must be used instead of the index, like "my_dict[a]".

One can bring items together by using the zip code. If one has two lists, they can zip them together to create tuples based on their location in the lists!

Finally, when printing or iterating through a lot of data (as in big data and deeplearning) generators can be helpful. With generators, the values are generated as they are requested instead of stored all at once, which can take time and power.

Also, decorators are a function that modifies a function and returns a function! It seemed confusing at first, but it is often helpful to wrap up a function in something else, to do something before and after another function, while still keeping the original function intact.

In[75]:
#example decorator to turn the items from earlier into strings

def make_string(func):
    def wrapper():
        items = func()
        return [f"Item: {str(item)}" for item in items]
    return wrapper

@make_string
def multiplication():
    return [1, 3, 10, 12]

print(multiplication())

['Item: 1', 'Item: 3', 'Item: 10', 'Item: 12']
